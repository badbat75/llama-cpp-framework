# CLAUDE.md

## Project overview

Windows-only PowerShell scripts to install prerequisites, configure, build, and run [llama.cpp](https://github.com/ggerganov/llama.cpp) with CUDA, Vulkan, and ROCm (HIP) support, optional Open WebUI frontend, and NSIS installer packaging. Not compatible with Linux or macOS.

## Structure

- `config.psd1` — central configuration (paths, build settings, runtime params). Generated by `01-configure.ps1`, not committed.
- `config.psd1.sample` — example config for reference.
- `common.ps1` — shared bootstrap: loads config, activates VS Dev Shell, sets up ROCm PATH.
- `00-install-prerequisites.ps1` — installs OpenSSL, CUDA, Vulkan SDK, AMD HIP via winget. Self-elevates via UAC.
- `01-configure.ps1` — auto-detects paths, verifies tools, writes `config.psd1`. Accepts `-LlamaCppDir` param (defaults to `.\build`).
- `02-build.ps1` — CMake configure + build with Ninja. Auto-clones/pulls llama.cpp repo. Uses sccache if available. Build output in `.\build\` (gitignored).
- `02-build-webui.ps1` — clones Open WebUI from GitHub, builds Svelte frontend (Node.js 18-22 via fnm), installs from source into Python 3.11/3.12 venv (`webui-venv/`, gitignored). Repo path from `config.psd1` (`OpenWebUIDir`). Skips frontend build if already done for current commit. Recreates venv on repo update.
- `03-package.ps1` — creates NSIS installer from build output + webui venv. Open WebUI installs to `C:\Program Files\Open WebUI` (separate from llama.cpp).
- `04-run.ps1` — launches `llama-server` from the installed distribution (reads install path from registry). Auto-starts Open WebUI on port 3000. Uses Windows Job Object to kill all child processes on exit.
- `llama-cpp.nsi.template` — NSIS installer template with placeholders (`@VERSION@`, `@STAGING_DIR@`, `@WEBUI_DIR@`, `@OUTPUT_FILE@`) replaced by `03-package.ps1`.
- `.webui_secret_key` — generated by Open WebUI at first run for JWT signing. Gitignored.

## Key conventions

- All scripts dot-source `common.ps1` (except `00-install-prerequisites.ps1`, `01-configure.ps1`, and `02-build-webui.ps1` which loads config directly).
- `common.ps1` activates VS Dev Shell and restores the working directory after.
- Config values use single-quoted strings in `.psd1` (no backslash escaping needed).
- Build output goes to `.\build\` (gitignored).
- The installer registers in `HKLM\Software\llama.cpp` (InstallDir, WebUIDir) and supports upgrades (uninstalls previous version first).
- sccache is auto-detected; cache stored in `.sccache/` (gitignored). Uses `SCCACHE_MAX_FRAME_LENGTH=104857600` for large GPU objects.
- Open WebUI dependencies are installed with relaxed pins (`==` → `>=`) to handle yanked packages on PyPI, then the package itself with `--no-deps`.
- Frontend build is tracked via `.build-commit` marker file in the Open WebUI repo root to skip unnecessary rebuilds.
