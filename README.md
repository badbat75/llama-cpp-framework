# llama.cpp-framework

> **Windows only** — all scripts are PowerShell (.ps1) and target Windows 10/11 exclusively.

PowerShell scripts to build and run [llama.cpp](https://github.com/ggerganov/llama.cpp) on Windows with multi-GPU backend support (CUDA, Vulkan, ROCm/HIP) and NSIS installer packaging. The chat UI built into `llama-server` is used as the frontend — no separate web UI is bundled.

## Prerequisites

- Windows 10/11
- [Visual Studio](https://visualstudio.microsoft.com/) with C++ workload
- [Git](https://git-scm.com/)

Optional (installed automatically by scripts as needed):
- [OpenSSL](https://slproweb.com/products/Win32OpenSSL.html)
- [CUDA Toolkit](https://developer.nvidia.com/cuda-toolkit)
- [Vulkan SDK](https://vulkan.lunarg.com/)
- [AMD HIP SDK (ROCm)](https://rocm.docs.amd.com/)
- [NSIS](https://nsis.sourceforge.io/) (for installer packaging, installed via winget if missing)

## Quick start

```powershell
# 1. Install / update the toolchain (elevates via UAC as needed).
#    Re-run any time to upgrade winget packages and pull llama.cpp.
.\00-install-prerequisites.ps1

# 2. Auto-detect paths and generate config-build.psd1
.\01-configure.ps1

# Or override the llama.cpp source clone location:
.\01-configure.ps1 -LlamaCppDir "C:\path\to\my\llama.cpp"

# 3. Build llama.cpp
.\02-build.ps1

# 4. Produce a Windows installer in dist\
.\03-package.ps1
```

## Layout

All build artifacts live under `build\` so a single `rm -rf build\` returns the repo to a clean state.

```
<repo>/
├── build/
│   ├── .sccache/           ← sccache compile cache
│   ├── config-build.psd1   ← build-time config (generated by 01-configure.ps1)
│   ├── llama-cpp.nsi       ← NSIS script (generated by 03-package.ps1, transient)
│   ├── llama.cpp/          ← llama.cpp source clone (LlamaCppDir)
│   ├── llama.cpp-cmake/    ← cmake binary dir (out-of-source build)
│   └── staging/            ← cmake --install staging area for NSIS
├── dist/                    ← final installer .exe
├── resources/               ← scripts shipped inside the installer
└── *.ps1                    ← top-level build/run/package scripts
```

## Configuration

Settings are split between build-time and runtime so the same scripts work both in-tree and from a system install.

| File | Where | Purpose |
|------|-------|---------|
| `config-build.psd1` | `build\` (gitignored) | Build-time paths and flags. Generated by `01-configure.ps1`. |
| `server.ini` | `%LOCALAPPDATA%\llama.cpp\config\` | Machine-wide llama-server runtime (port, host, threads, mlock, models dir, max simultaneous models). Single `[Server]` section. Generated by `resources\config-server.ps1`. |
| `presets.ini` | `%LOCALAPPDATA%\llama.cpp\config\` | Per-model presets: one `[<model-id>]` section each with model path, ctx size, GPU layers, batch sizes, cache types, flash-attn, sampling, MoE expert offload, chat template. Consumed directly by `llama-server --models-preset`. Generated by `resources\config-model.ps1`. |

`config-build.psd1` keys:

| Section | Keys |
|---------|------|
| Paths | `LlamaCppDir`, `OpenSSLDir`, `HipPath`, `VsDevShell`, `CacheDir` |
| Build | `GpuTargets`, `BuildType`, `CCompiler`, `CxxCompiler`, `MarchFlags`, `BuildJobs` |

Runtime configs are written on first launch (or by the NSIS install-time page). They can be regenerated at any time via the Start Menu shortcuts (*Configure llama-server*, *Configure model*) shipped with the installer — each shortcut runs the interactive writer once and the window closes when you're done.

## Scripts

| Script | Description |
|--------|-------------|
| `00-install-prerequisites.ps1` | Idempotent toolchain bootstrapper. Installs missing winget packages (PowerShell 7+, OpenSSL, NSIS) and upgrades present ones in a single UAC-elevated session; pulls llama.cpp source when cloned; flags CUDA / Vulkan / HIP SDKs for manual install; reports version changes and recommends a rebuild only when llama.cpp source moved. |
| `01-configure.ps1` | Detects environment, verifies tools, writes `config-build.psd1`. Accepts `-LlamaCppDir`. |
| `02-build.ps1` | CMake configure (out-of-source into `build\llama.cpp-cmake\`) + Ninja build. Auto-clones llama.cpp into `build\llama.cpp\`. Uses sccache when available. |
| `03-package.ps1` | Stages from `build\llama.cpp-cmake\` into `build\staging\`, then runs NSIS to produce `dist\llama_cpp-<version>-win64-setup.exe`. |
| `common.ps1` | Shared bootstrap: loads `$cfg` from `config-build.psd1`, prepends ROCm `bin\` to PATH, exposes `Enable-VsDevShell` as a function. VS Dev Shell activation is **opt-in** — only build/package scripts call it, runtime scripts don't pay the startup cost. |
| `resources\run-model.ps1` | Runtime entry point shipped with the installer. Reads `server.ini` + `presets.ini`; auto-invokes the config writers on first launch. Accepts `-ServerExe` to override the llama-server binary location. |
| `resources\config-server.ps1` / `config-model.ps1` | Interactive config writers for the user-scope INI files (`server.ini` / `presets.ini`). Each supports `-NonInteractive` so NSIS can call them at install time. |
| `resources\common-functions.ps1` | Shared INI parser/writer for `server.ini` (`Read-ServerIni`, `Set-ServerIniField`). Dot-sourced by all three runtime scripts. |

## Packaging

`03-package.ps1` produces a Windows installer at `dist\llama_cpp-<version>-win64-setup.exe`. The installer:

- Installs llama.cpp binaries to `C:\Program Files\llama.cpp\`.
- Optionally adds `bin\` to the system PATH.
- Creates Start Menu shortcuts (`llama.cpp`, *Configure llama-server*, *Configure model*, *Uninstall*) targeting `pwsh.exe` (PowerShell 7+).
- Prompts for llama-server port + bind + thread counts + models folder during install (machine-wide knobs); per-model parameters (ctx size, GPU layers, batch sizes, etc.) are collected lazily on first launch via `config-model.ps1`.
- Supports upgrades (silently uninstalls the previous version first).
- Includes a full uninstaller in Add/Remove Programs. By default it preserves `%LOCALAPPDATA%\llama.cpp\` (configs, logs) across uninstalls; the user can opt in to a full wipe.

NSIS is installed automatically via winget if not already present.
