# llama.cpp-framework

> **Windows only** — all scripts are PowerShell (.ps1) and target Windows 10/11 exclusively.

PowerShell scripts to build and run [llama.cpp](https://github.com/ggerganov/llama.cpp) on Windows with multi-GPU backend support (CUDA, Vulkan, ROCm/HIP), optional [Open WebUI](https://github.com/open-webui/open-webui) frontend, and NSIS installer packaging.

## Prerequisites

- Windows 10/11
- [Visual Studio](https://visualstudio.microsoft.com/) with C++ workload
- [Git](https://git-scm.com/)

Optional (installed automatically by scripts as needed):
- [OpenSSL](https://slproweb.com/products/Win32OpenSSL.html)
- [CUDA Toolkit](https://developer.nvidia.com/cuda-toolkit)
- [Vulkan SDK](https://vulkan.lunarg.com/)
- [AMD HIP SDK (ROCm)](https://rocm.docs.amd.com/)
- Python 3.11 or 3.12 (for Open WebUI, installed via winget if missing)
- Node.js 18-22 (for Open WebUI frontend build, managed via fnm if system version is incompatible)
- [NSIS](https://nsis.sourceforge.io/) (for installer packaging, installed via winget if missing)

## Quick start

```powershell
# 1. Install dependencies (elevates via UAC as needed)
.\00-install-prerequisites.ps1

# 2. Auto-detect paths and generate config-build.psd1
.\01-configure.ps1

# Or override the llama.cpp source clone location:
.\01-configure.ps1 -LlamaCppDir "C:\path\to\my\llama.cpp"

# 3. Build llama.cpp
.\02-build.ps1

# 3b. (Optional) Build Open WebUI from source
.\02-build-webui.ps1

# 4. Run from the dev build (no install needed)
.\04-run.ps1                # llama-server only
.\04-run.ps1 -WithWebUI     # llama-server + Open WebUI on http://localhost:8180

# 5. (Optional) Produce a Windows installer in dist\
.\03-package.ps1

# 6. (Optional) Keep tooling + sources up to date
.\05-update-framework.ps1
```

## Layout

All build artifacts live under `build\` so a single `rm -rf build\` returns the repo to a clean state.

```
<repo>/
├── build/
│   ├── llama.cpp/          ← llama.cpp source clone (LlamaCppDir)
│   ├── llama.cpp-cmake/    ← cmake binary dir (out-of-source build)
│   ├── open-webui/         ← Open WebUI source clone (OpenWebUIDir)
│   ├── webui-venv/         ← Python venv with open-webui installed
│   └── staging/            ← cmake --install staging area for NSIS
├── dist/                    ← final installer .exe
├── .sccache/                ← sccache compile cache (survives clean wipes)
├── resources/               ← scripts shipped inside the installer
└── *.ps1                    ← top-level build/run/package scripts
```

## Configuration

Settings are split between build-time and runtime so the same scripts work both in-tree and from a system install.

| File | Where | Purpose |
|------|-------|---------|
| `config-build.psd1` | repo root (gitignored) | Build-time paths and flags. Generated by `01-configure.ps1`. |
| `server.psd1` | `%LOCALAPPDATA%\llama.cpp\config\` | Machine-wide llama-server runtime (port, host, threads, mlock, active model pointer). Generated by `resources\config-server.ps1`. |
| `models\<id>.psd1` | `%LOCALAPPDATA%\llama.cpp\config\` | Per-model knobs: model path, ctx size, GPU layers, batch sizes, cache types, flash-attn, sampling, MoE expert offload, chat template. Generated by `resources\config-model.ps1`. |
| `webui.psd1` | `%LOCALAPPDATA%\llama.cpp\config\` | Open WebUI bind/port. Generated by `resources\config-webui.ps1`. |

`config-build.psd1` keys:

| Section | Keys |
|---------|------|
| Paths | `LlamaCppDir`, `OpenWebUIDir`, `OpenSSLDir`, `HipPath`, `VsDevShell`, `CacheDir` |
| Build | `GpuTargets`, `BuildType`, `CCompiler`, `CxxCompiler`, `MarchFlags`, `BuildJobs` |

Runtime configs are written on first launch (or by the NSIS install-time pages). They can be regenerated at any time via the Start Menu shortcuts (*Configure llama-server*, *Configure model*, *Configure Open WebUI*) shipped with the installer — each shortcut runs the interactive writer once and the window closes when you're done.

## Scripts

| Script | Description |
|--------|-------------|
| `00-install-prerequisites.ps1` | Installs PowerShell 7+ and OpenSSL via winget (UAC self-elevation); flags CUDA / Vulkan / HIP SDKs for manual install. |
| `01-configure.ps1` | Detects environment, verifies tools, writes `config-build.psd1`. Accepts `-LlamaCppDir`. |
| `02-build.ps1` | CMake configure (out-of-source into `build\llama.cpp-cmake\`) + Ninja build. Auto-clones llama.cpp into `build\llama.cpp\`. Uses sccache when available. |
| `02-build-webui.ps1` | Clones Open WebUI, builds the Svelte frontend, installs from source into `build\webui-venv\`. Reuses a previous frontend build if the commit hasn't changed. |
| `03-package.ps1` | Stages from `build\llama.cpp-cmake\` into `build\staging\`, then runs NSIS to produce `dist\llama_cpp-<version>[-webui-<version>]-win64-setup.exe`. |
| `04-run.ps1` | Dev-time launcher. Thin wrapper around `resources\run-model.ps1` pointing at `build\llama.cpp-cmake\bin\llama-server.exe`. Pass `-WithWebUI` to also launch Open WebUI from `build\webui-venv\` in the background (killed via `taskkill /T` on exit). |
| `05-update-framework.ps1` | `winget upgrade` for tracked tooling + `git pull --ff-only` on llama.cpp and Open WebUI. Reports what moved and suggests a rebuild only when sources actually changed. |
| `common.ps1` | Shared bootstrap: loads `$cfg` from `config-build.psd1`, prepends ROCm `bin\` to PATH, exposes `Enable-VsDevShell` as a function. VS Dev Shell activation is **opt-in** — only build/package scripts call it, runtime scripts don't pay the startup cost. |
| `resources\run-model.ps1` | Runtime entry point shipped with the installer. Reads `server.psd1` + active model config; auto-invokes the config writers on first launch. Accepts `-ServerExe` so the dev wrapper can point it at `build\llama.cpp-cmake\bin\`. |
| `resources\start-webui.ps1` | Standalone Open WebUI launcher. Probes whether llama-server is up and seeds `OPENAI_API_BASE_URL`. Accepts `-WebUIExe` for dev mode. |
| `resources\config-server.ps1` / `config-model.ps1` / `config-webui.ps1` | Interactive config writers for the user-scope `.psd1` files. Each supports `-NonInteractive` so NSIS can call them at install time. |

## Open WebUI

`02-build-webui.ps1` builds [Open WebUI](https://github.com/open-webui/open-webui) from source:

1. Clones/pulls the repo (path from `OpenWebUIDir` in `config-build.psd1`, default `.\build\open-webui`).
2. Ensures compatible Node.js 18-22 (installs via [fnm](https://github.com/Schniz/fnm) if needed).
3. Builds the Svelte frontend (`npm install` + `npm run build`); skipped if the marker file matches the current commit.
4. Creates a Python 3.11/3.12 venv at `build\webui-venv\` and installs Open WebUI from source. Requirements use relaxed pins (`==` → `>=`) to tolerate yanked PyPI versions; the package itself is installed with `--no-deps`.
5. The venv is wiped and recreated when the upstream repo moves.

`04-run.ps1 -WithWebUI` launches Open WebUI in the background and kills the full process tree on exit (`taskkill /F /T`). Open WebUI writes its database, vector store, and uploads to `%LOCALAPPDATA%\llama.cpp\data\` (via `DATA_DIR`) so nothing lands inside Program Files.

## Packaging

`03-package.ps1` produces a Windows installer at `dist\llama_cpp-<version>[-webui-<version>]-win64-setup.exe`. The installer:

- Installs llama.cpp binaries to `C:\Program Files\llama.cpp\`.
- Optionally installs Open WebUI to `C:\Program Files\Open WebUI\` (self-contained Python venv; `relocate-venv.py` rewrites launcher shebangs post-copy).
- Optionally adds `bin\` to the system PATH.
- Creates Start Menu shortcuts (`llama.cpp`, *Configure llama-server*, *Configure model*, *Start Open WebUI*, *Configure Open WebUI*, *Uninstall*) targeting `pwsh.exe` (PowerShell 7+).
- Prompts for llama-server port + bind during install (machine-wide knobs); per-model parameters (ctx size, GPU layers, batch sizes, etc.) are collected lazily on first launch via `config-model.ps1`.
- Supports upgrades (silently uninstalls the previous version first).
- Includes a full uninstaller in Add/Remove Programs. By default it preserves `%LOCALAPPDATA%\llama.cpp\` (configs, chat history, logs) across uninstalls; the user can opt in to a full wipe.

NSIS is installed automatically via winget if not already present.
