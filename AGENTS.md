# Project guide

## Project overview

Windows-only PowerShell scripts to install prerequisites, configure, build, and run [llama.cpp](https://github.com/ggerganov/llama.cpp) with CUDA, Vulkan, and ROCm (HIP) support, optional Open WebUI frontend, and NSIS installer packaging. Not compatible with Linux or macOS.

## Structure

- `config.psd1` — central configuration (paths, build settings, runtime params). Generated by `01-configure.ps1`, not committed.
- `config.psd1.sample` — example config for reference.
- `common.ps1` — shared bootstrap: loads config, activates VS Dev Shell, sets up ROCm PATH.
- `00-install-prerequisites.ps1` — installs PowerShell 7+ (`pwsh`, required by the runtime shortcuts) and OpenSSL via winget; flags CUDA / Vulkan SDK / AMD HIP for manual install. Self-elevates via UAC; combines all winget installs into a single elevated session.
- `01-configure.ps1` — auto-detects paths, verifies tools, writes `config.psd1`. Accepts `-LlamaCppDir` param (defaults to `.\build`).
- `02-build.ps1` — CMake configure + build with Ninja. Auto-clones/pulls llama.cpp repo. Uses sccache if available. Build output in `.\build\` (gitignored).
- `02-build-webui.ps1` — clones Open WebUI from GitHub, builds Svelte frontend (Node.js 18-22 via fnm), installs from source into Python 3.11/3.12 venv (`webui-venv/`, gitignored). Repo path from `config.psd1` (`OpenWebUIDir`). Skips frontend build if already done for current commit. Recreates venv on repo update.
- `03-package.ps1` — creates NSIS installer from build output + webui venv. Open WebUI installs to `C:\Program Files\Open WebUI` (separate from llama.cpp).
- `05-update-framework.ps1` — maintenance script: `winget upgrade` for tracked tooling (PowerShell, OpenSSL, Python 3.12, fnm, NSIS), `git pull --ff-only` on llama.cpp and Open WebUI repos, and a final report of what changed. Recommends `02-build.ps1` / `02-build-webui.ps1` / `03-package.ps1` only when llama.cpp or Open WebUI sources moved (the only deps that get baked into the shipped binary). Manual SDKs (CUDA, Vulkan, ROCm) are not touched — only flagged.
- `04-run.ps1` — dev-time launcher: dot-sources `common.ps1` (VS Dev Shell + ROCm DLLs on PATH). Reads the same split user configs as the installed runtime (server.psd1 + active model). Pass `-WithWebUI` to also launch Open WebUI (sets `DATA_DIR` to `%LOCALAPPDATA%\llama.cpp\data` so ChromaDB / `webui.db` land in a writable location).
- `resources/run-model.ps1` — runtime entry point shipped with the installer (main "llama.cpp" Start Menu shortcut). Reads `server.psd1` + `models\<ActiveModel>.psd1`; auto-invokes `config-server.ps1` / `config-model.ps1` if missing. Starts `llama-server` only. Tees stdout+stderr to `%LOCALAPPDATA%\llama.cpp\logs\llama-server.log`. CWD is forced to `%LOCALAPPDATA%\llama.cpp` so first-run files don't land in Program Files.
- `resources/start-webui.ps1` — standalone Open WebUI launcher (Start Menu shortcut "Start Open WebUI"). Reads `webui.psd1` for its own bind/port and `server.psd1` for the llama-server endpoint to probe. Sets `DATA_DIR` to `%LOCALAPPDATA%\llama.cpp\data` (writable per-user; required because Open WebUI's default falls back inside Program Files and crashes ChromaDB). Tees stdout+stderr to `%LOCALAPPDATA%\llama.cpp\logs\open-webui.log`. Doesn't auto-start llama-server.
- `resources/config-server.ps1` — llama-server config writer. Writes `%LOCALAPPDATA%\llama.cpp\config\server.psd1`. Holds only machine-wide knobs (Port, Hostname, Mlock, Threads); model-dependent params (CtxSize, GpuLayers, Parallel, CacheTypeK/V, FlashAttn) live in `models\<id>.psd1`. NSIS calls it with `-NonInteractive -Port -Hostname` from the install-time page. Preserves `ModelsDir` / `ActiveModel` set by config-model.ps1. Re-runnable via "Configure llama-server" Start Menu shortcut.
- `resources/config-webui.ps1` — Open WebUI config writer. Writes `webui.psd1` (Port + Hostname). Same interactive / `-NonInteractive` dual mode as config-server.ps1.
- `resources/config-model.ps1` — per-model interactive config builder. Lists `.gguf` files under `ModelsDir` (marks already-configured ones with `*`), prompts for every model param (`CtxSize`, `GpuLayers`, `Parallel`, `CacheTypeK/V`, `FlashAttn`, `Jinja`, `NCpuMoe`, sampling, `ChatTemplateKwargs`), writes `models\<id>.psd1`, and updates `server.psd1` with `ActiveModel = '<id>'`. Re-runnable via "Configure model" Start Menu shortcut to switch active model or refresh params.
- `llama-cpp.nsi.template` — NSIS installer template with placeholders (`@VERSION@`, `@STAGING_DIR@`, `@WEBUI_DIR@`, `@OUTPUT_FILE@`) replaced by `03-package.ps1`. Has two custom nsDialogs pages (server config: Port / Bind; webui config: Port / Bind, conditional on SecWebUI) that drive `config-server.ps1 -NonInteractive` and `config-webui.ps1 -NonInteractive` post-install to write the PSD1 files. Model-dependent params (CtxSize, GpuLayers, Parallel, CacheTypeK/V, FlashAttn, sampling, etc.) are NOT prompted at install — created lazily per-model by config-model.ps1 on first run. Start Menu shortcuts target `pwsh.exe` (PowerShell 7+).
- `.webui_secret_key` — generated by Open WebUI at first run for JWT signing. Gitignored.

## Key conventions

- All scripts dot-source `common.ps1` (except `00-install-prerequisites.ps1`, `01-configure.ps1`, and `02-build-webui.ps1` which loads config directly).
- `common.ps1` activates VS Dev Shell and restores the working directory after.
- Config values use single-quoted strings in `.psd1` (no backslash escaping needed).
- Build output goes to `.\build\` (gitignored).
- The installer registers in `HKLM\Software\llama.cpp` (InstallDir, WebUIDir) and supports upgrades (uninstalls previous version first). User runtime state lives under `%LOCALAPPDATA%\llama.cpp\`: `config\server.psd1` (llama-server runtime + ModelsDir + ActiveModel pointer), `config\webui.psd1` (Open WebUI bind/port), `config\models\<id>.psd1` (per-model: model path, Jinja, NCpuMoe, sampling), `data\` (Open WebUI's `webui.db` / `vector_db/` / `uploads/`), `logs\` (`llama-server.log`, `open-webui.log`). The uninstaller asks (defaulting to No) whether to delete the whole tree, so it survives reinstall/upgrade by default.
- sccache is auto-detected; cache stored in `.sccache/` (gitignored). Uses `SCCACHE_MAX_FRAME_LENGTH=104857600` for large GPU objects.
- Open WebUI dependencies are installed with relaxed pins (`==` → `>=`) to handle yanked packages on PyPI, then the package itself with `--no-deps`.
- Frontend build is tracked via `.build-commit` marker file in the Open WebUI repo root to skip unnecessary rebuilds.
